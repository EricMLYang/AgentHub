# Prompt: 建立新的 InProgress 專案檔案

## 使用方式

### 1. 選擇要開始的 idea
從 `10_idea/` 資料夾選擇一個你要開始執行的 idea。

### 2. 複製以下 prompt 並填入資訊

```
請幫我建立一個新的 InProgress 專案追蹤檔案。

**原始 Idea**: 10_idea/[idea 檔案名稱]
**負責人**: [你的名字]
**開始日期**: [YYYY-MM-DD]
**初始狀態**: [Planning / Building / Testing]

請根據原始 idea 的內容，產生一個專案追蹤檔案，包含：

1. **基本資訊**：關聯原始 idea、開始日期、目前狀態
2. **執行計畫**：根據 idea 的複雜度調整 Phase，保持簡單
3. **第一週的開發紀錄**：包含本週計畫（根據 idea 內容）
4. **驗收標準檢核**：直接從原始 idea 複製過來

注意：
- 保持簡單，不用填寫太多空白區塊
- 技術決策、測試、風險等區塊先留空，之後再補
- 重點是能快速開始追蹤進度

檔案要儲存到：20_inProgress/[與 idea 相同的檔名]
```

### 3. Copilot 會產生檔案內容

檢查產生的內容：
- [ ] 是否正確連結到原始 idea？
- [ ] 執行計畫是否合理？
- [ ] 驗收標準是否完整？

### 4. 建立檔案並開始追蹤

把內容儲存到 `20_inProgress/` 資料夾，開始你的專案！

---

## 範例

### 輸入

```
請幫我建立一個新的 InProgress 專案追蹤檔案。

**原始 Idea**: 10_idea/idea-01-automated-analytics-reporting.md
**負責人**: @Eric
**開始日期**: 2026-01-08
**初始狀態**: Planning

請根據原始 idea 的內容，產生一個專案追蹤檔案。
```

### Copilot 產出

檔案：`20_inProgress/idea-01-automated-analytics-reporting.md`

```markdown
# [InProgress] 自動化數據分析報告 Agent

> 📌 關聯 Idea: [idea-01-automated-analytics-reporting](../10_idea/idea-01-automated-analytics-reporting.md)  
> 🚀 開始日期: 2026-01-08  
> 👤 負責人: @Eric  
> 📊 目前狀態: `🟡 Planning`

---

## 📋 執行計畫

### Phase 1: 研究與設計
- [ ] 完成技術調研（LLM 框架、數據連接）
- [ ] 確認 Databricks 存取權限
- [ ] 設計報告生成流程
- [ ] 建立簡單 POC

### Phase 2: 核心開發
- [ ] 實作數據提取模組
- [ ] 實作環比/同比計算
- [ ] 整合 LLM 生成洞察
- [ ] 基本測試

### Phase 3: 整合與部署
- [ ] 整合 Slack 通知
- [ ] 建立排程機制
- [ ] 使用者測試
- [ ] 撰寫文件

---

## 📝 開發紀錄

### Week 2026-W02 (01/08 - 01/12) - @Eric

**本週計畫**
- 調研 LangChain vs 直接使用 OpenAI API
- 申請 Databricks token
- 設計系統架構

**需要協助**
無

---

## ✅ 驗收標準檢核

根據原始 idea：
- [ ] 報告包含所有必要章節（概況、環比、細分、異常、洞察、來源）
- [ ] 數值與來源數據一致（對帳檢核通過）
- [ ] 所有數據引用必須標註來源
- [ ] 異常超過閾值時會自動標記並說明
- [ ] 按時產出（準時率 > 95%）
- [ ] 洞察與行動建議段落需經人工審核（HITL）

---

## 📌 備註

（其他區塊如技術決策、測試等，之後再補充）
```

---

## 小提示

**從簡單開始**：一開始不用填寫所有區塊，先建立檔案就好

**參考 idea**：讓 Copilot 讀取原始 idea，自動產生合適的執行計畫

**調整 Phase**：根據專案大小調整階段，小專案 2-3 個 Phase 就夠

**標記負責人**：如果是團隊協作，記得加上負責人

---

## 完整模板參考

如果需要完整的模板結構，可以參考 `20_inProgress/idea-01-automated-analytics-reporting.md` 這個範例檔案。

完整模板包含：
- 📋 執行計畫（4 個 Phase）
- 📝 開發紀錄（週報格式）
- 🔧 技術決策紀錄
- 🧪 測試 & 驗證
- 💡 學習與調整
- 🔗 相關資源
- 📊 成本與效益追蹤
- 🚧 風險與阻礙
- ✅ 完成檢核清單

**但記住**：一開始不用全部填寫，保持簡單，隨著專案進行再逐步補充！
