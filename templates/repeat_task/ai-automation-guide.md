# AI 化建議欄位說明

此欄位用於收集「可透過 AI 自動化」的子任務建議，協助團隊評估哪些重複任務適合轉化為 Agent。

---

## 填寫格式

**AI 化建議**：
- [ ] 子任務名稱 — 可行性 🟢/🟡/🔴 — 技術路徑（Workflow / RAG / Agent）— 預估省時 X 小時/次

---

## 可行性指標

- 🟢 **高可行性**：規則明確、可驗證、現有工具可實現
- 🟡 **中可行性**：需要一些人工判斷或 LLM 輔助，但大部分可自動化
- 🔴 **低可行性**：高度依賴人類經驗判斷、創意、或複雜決策

---

## 技術路徑說明

- **Workflow（工作流）**：步驟固定、無需 AI 判斷（如：自動 ETL、排程任務）
- **RAG（檢索增強生成）**：需要查閱文件並回答（如：FAQ 機器人、文件搜尋）
- **Agent（智能代理）**：需要多步推理、調用工具、主動決策（如：診斷異常、多步操作）

---

## 範例

### 範例 1：數據分析類
**原始任務**：每週手動整理多來源數據並產出報告（45 分鐘/次）

**AI 化建議**：
- [x] 數據提取與清洗 — 🟢 — Workflow — 省 15 分鐘
- [x] 計算環比/同比指標 — 🟢 — Workflow — 省 10 分鐘
- [ ] 撰寫洞察與建議 — 🟡 — RAG + LLM — 省 15 分鐘（需人工審核）
- [ ] 異常診斷與追因 — 🟡 — Agent — 省 5 分鐘（複雜情況仍需人工）

**總結**：高可行性，建議優先開發 Workflow 部分 → 再加入 LLM 輔助敘事。

### 範例 2：系統開發類
**原始任務**：根據設計稿產出 Vue 元件（2 小時/個）

**AI 化建議**：
- [ ] 讀取設計稿並產出元件結構 — 🟡 — Agent (Vision + Code Gen) — 省 1 小時
- [ ] 產出對應的 API Swagger 文件 — 🟢 — Workflow + Template — 省 30 分鐘
- [ ] 執行 lint 與自動修正 — 🟢 — Workflow — 省 15 分鐘
- [ ] 商業邏輯實作 — 🔴 — 仍需人工（AI 只能輔助基本 CRUD）

**總結**：中可行性，可先做「元件框架生成」+ lint 自動化，商業邏輯仍需人工。

---

## 使用方式

1. 在各大類檔案（01_日常業務、02_數據分析...）的每個任務下方，新增「AI 化建議」段落
2. 團隊成員可以在討論時逐步填寫、更新可行性評估
3. 當某個任務的 AI 化建議成熟後，可轉為正式 Idea 並進入評估流程
