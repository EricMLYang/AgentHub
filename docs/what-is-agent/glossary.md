# Agent 詞彙表（Glossary）

> 簡短定義常用術語，避免溝通誤解

---

## Agent
一個可以自主決策、調用工具、根據環境調整行為來完成目標的 AI 系統。  
**關鍵**：有決策能力、可組合工具、行為不固定

---

## Tool
Agent 可以呼叫的「函式」或「API」，例如：搜尋引擎、執行程式碼、讀寫檔案、發送通知等。  
**關鍵**：Agent 透過 Tool 與外部世界互動

---

## Workflow
預先定義好步驟與分支的自動化流程，通常用有向圖 (DAG) 表示。  
**關鍵**：路徑固定、可預測、適合已知業務流程

---

## RAG (Retrieval-Augmented Generation)
先從知識庫「檢索」相關內容，再讓 LLM「生成」回答的技術。  
**關鍵**：適合 Q&A，但不是 Agent（沒有決策與工具組合能力）

---

## Guardrail
限制 Agent 行為的「護欄」，例如：不能刪除資料庫、不能發送未經審查的 email。  
**關鍵**：確保 Agent 不會做出危險或不當的行為

---

## Acceptance Criteria
用來驗證 Agent 是否「完成任務」的標準，必須是可觀察、可否定的條件。  
**關鍵**：要具體，不能是「做得好」或「優化生產力」這種模糊說法

---

## Prompt
給 LLM 的指令，通常包含任務描述、範例、格式要求等。  
**關鍵**：Agent 的「大腦」依賴好的 Prompt 來理解目標

---

## Context Window
LLM 一次能「看到」的文字長度上限（例如 128K tokens）。  
**關鍵**：影響 Agent 能處理多少資訊、多長的對話歷史

---

## Fallback
當 Agent 無法處理時的「降級方案」，例如：轉人工、發警告、記錄錯誤。  
**關鍵**：確保系統有韌性，不會因為 Agent 失敗而整個掛掉

---

## Shaping
在執行前先「釐清需求」的過程，確保 Input / Output / Acceptance 都清楚。  
**關鍵**：避免做出沒人要的 Agent
