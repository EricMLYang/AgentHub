## ▋ 第一階段：策略定位評估 (Strategic Fit)

在進入功能細節前，先判斷該 Idea 在市場與商業模式上的「高度」。

* **是否屬於 Vertical AI？** * Idea 是否針對特定行業（如零售 RMN、醫療、法律）？
* 是否能像「船」一樣，隨著通用模型（海水）水位上升而持續浮在水面，具備獨特的利基？


* **從 SaaS 轉向 RaaS (Results as a Service) 的潛力：**
* 產品能否從「提供工具」進化為「交付成果」（Job Done）？
* 商業模式是否能按「成功解決件數」或「達成指標」收費（Selling the Work）？


* **數據飛輪與護城河：**
* 是否能透過 **Function Calling** 串接業務系統，自動獲取「最終結果」作為 Ground Truth（地真數據）進行自我標註與微調？
* 是否利用了 Databricks Lakehouse 處理非結構化（影像/人流）與結構化（POS/交易）數據的優勢？

## ▋ 第二階段：任務選型與技術路徑 (Task Selection)

並非所有功能都適合 Agent。透過以下矩陣判斷技術路徑：

### 1. 技術選型矩陣

| 技術類型 | 適用場景 | 優點 | 缺點 |
| --- | --- | --- | --- |
| **純程式碼** | 輸入固定、規則死、低延遲、高稽核 | 最便宜、穩定、好測試 | 無法處理模糊需求 |
| **工作流 (Workflow)** | 步驟清楚、分支有限、例外可控 | 可控、可追溯 | 遇到新花樣會失效 |
| **RAG (問答)** | 查資料、讀文件、回答問題 | 提升理解效率 | 不會主動「做事」 |
| **AI Agent** | **高度變動輸入、多步推理、需自主決策行動** | **具備主見、全責交付** | **成本高、延遲高、難治理** |

### 2. F.D.A 框架 (快速篩選高價值題目)

* **F (Friction) 摩擦點：** 是否為高頻、重複且目前需「人工審核」的環節？
* **D (Data Nexus) 數據交匯：** 是否處於非結構化與結構化數據的交叉點（如：影像人流 + POS 銷售）？處理數據交叉後的「決策」最值錢。
* **A (Actionable Outcome) 可執行成果：** 是否能從「給圖表」進化到「交付工作成果」（如：自動產出廣告結案報告或自動優化排程）？

## ▋ 第三階段：Agent 化適配評分表 (The Scorecard)

針對特定任務進行評分，分數越高越適合 Agent 化。

### 核心評估維度：

1. **規則明確性 (Clarity)：** 任務步驟是否可被形式化拆解？（雖然輸入模糊，但判斷標準須明確）。
2. **結果可驗證性 (Verifiability)：** 是否有明確的「成功/失敗」判準？是否有類似 Compiler 或 Unit Test 的自動驗證機制（Golden Set）？
3. **回饋延遲性 (Latency)：** 獲取對錯訊號的時間是否夠短？（秒級或分鐘級的回饋最適合 Agent 進化）。
4. **重複性與頻率 (Frequency)：** 任務是否大量存在？
5. **狀態空間可模擬性 (Simulatable)：** 該領域能否在數位環境中（如沙盒）精準模擬試錯而不影響現實？
6. **數據標註可得性：** 是否能從系統日誌自動產生標籤（如「播放成功」、「對帳無誤」）？

---

## ▋ 第四階段：Databricks 專屬架構核對

身為 Databricks-first 團隊，Idea 必須通過「三層架構」檢驗：

1. **數據基礎層：** 數據是否已在 Unity Catalog 中完成治理？是否具備完整數據血緣 (Lineage)？
2. **策略分析層：** 是否有現成的 Pipeline 將數據轉為「金級表」？是否有微調過的領域專用模型（如針對零售術語的 Embedding）？
3. **互動 Agent 層：** * **人機協作 (HITL)：** 高風險動作是否設計了人工審核閘口？
* **工具契約 (Tools)：** 是否已將 SQL 或 Python 函數註冊為 Unity Catalog Functions 供 Agent 調用？



---

## ▋ 評估標準總結清單 (Checklist)

* [ ] **它是否有主見 (Opinionated)？** 產品是否替用戶做了前置決策，而非只給一堆開關。
* [ ] **它是否解決了 95% 的周邊工程？** 包含 Context 管理、Memory 設計與評估體系。
* [ ] **它是否從「深且窄」開始？** 任務是否窄到能徹底優化、深到值得客戶為「成果」付錢。
* [ ] **它的失敗成本是否可控？** 失敗後是否有自動回填機制告知 Agent「此路不通」。
